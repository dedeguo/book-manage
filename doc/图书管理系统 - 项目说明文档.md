# 图书管理系统 - 项目说明文档

## 1. 项目概述

### 1.1 项目背景
图书管理系统是一个典型的信息管理系统，本系统作为学生课程实验项目，旨在让学生掌握软件开发的基本流程，包括需求分析、系统设计、编码实现和测试部署。

### 1.2 系统目标
- 提供便捷的图书查询和借阅功能
- 简化图书管理流程
- 为不同角色（普通用户、管理员）提供差异化功能
- 支持多复本管理（同一图书可有多本可借）

### 1.3 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 已注册的学生/教师，可浏览和借阅图书 |
| 管理员 | 系统管理员，可管理图书和用户 |

**管理员初始化**：系统预设一个默认管理员账号（admin/123456），首次登录后建议修改密码

## 2. 功能需求

### 2.1 用户模块

#### 2.1.1 注册
- 用户填写学号/工号、姓名、密码、邮箱等信息进行注册
- 学号/工号唯一，密码需要加密存储
- 系统根据学号/工号格式自动判断用户类型（学生/教师）
- 注册成功后自动跳转到登录页面

#### 2.1.2 登录
- 用户使用用户名和密码登录
- 登录成功后根据角色跳转到不同首页
- 支持退出登录功能

#### 2.1.3 个人信息
- 查看个人信息（学号/工号、姓名、用户类型、邮箱、注册时间等）
- 修改密码
- 修改邮箱

### 2.2 图书模块（所有用户可见）

#### 2.2.1 图书浏览
- 以列表形式展示所有图书
- 显示图书封面、书名、作者、出版社、简介等信息
- 支持按书名、作者模糊搜索
- 支持按分类筛选
- 显示图书可借复本数

#### 2.2.2 图书详情
- 查看图书详细信息
- 显示图书状态：可借复本数 / 总复本数（如：3/5 可借）

### 2.3 借阅模块（普通用户）

#### 2.3.1 我的借阅
- 查看当前借阅的图书列表
- 查看每本书的借阅日期和应还日期
- 显示借阅状态（借阅中/已归还）
- 可查看历史借阅记录（已归还的图书）

#### 2.3.2 借阅图书
- 在图书详情页点击"借阅"按钮
- 每人最多同时借阅 5 本书（借阅中状态）
- 借阅期限 30 天
- 只有当图书可借复本数 > 0 时才能借阅

#### 2.3.3 归还图书
- 在"我的借阅"页面点击"归还"按钮
- 归还后图书可借复本数 +1

### 2.4 图书管理模块（管理员）

#### 2.4.1 新增图书
- 填写图书信息：书名、作者、出版社、ISBN、简介、封面 URL
- 可设置复本数（默认 1）
- 新增后图书可借复本数 = 总复本数

#### 2.4.2 编辑图书
- 修改已有图书信息
- 不能修改总复本数（避免借阅记录错乱）

#### 2.4.3 删除图书
- 只能删除无人借阅的图书（可借复本数 = 总复本数）
- 有借阅记录的图书不能删除（可借复本数 < 总复本数）

### 2.5 用户管理模块（管理员）

#### 2.5.1 用户列表
- 查看所有注册用户
- 显示学号/工号、姓名、用户类型、邮箱、注册时间

#### 2.5.2 删除用户
- 删除违规用户账号
- 有借阅在身的用户不能删除
- 管理员账号不能删除（操作列显示"-"）

### 2.6 借阅管理模块（管理员）

#### 2.6.1 借阅记录列表
- 查看所有用户的借阅记录
- 显示：借阅人（学号/姓名）、图书名称、借阅日期、应还日期、归还日期、状态
- 支持按学号/工号、姓名、图书名称搜索
- 支持按借阅状态筛选（借阅中/已归还）

#### 2.6.2 手动归还
- 管理员可代为归还图书（如图书损坏、丢失等特殊场景）
- 操作后更新借阅记录状态和图书可借复本数

#### 2.6.3 超期记录查看
- 查看超期未还的借阅记录
- 显示超期天数

## 3. 数据需求

### 3.1 数据实体

#### 用户表 (users)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| username | varchar | 用户名（学号/工号，唯一） |
| name | varchar | 姓名 |
| password | varchar | 密码（加密） |
| email | varchar | 邮箱 |
| role | varchar | 角色（user/admin） |
| user_type | varchar | 用户类型（student/teacher/admin），根据学号/工号自动判断） |
| created_at | datetime | 注册时间 |

> **用户类型判断规则**：
> - `role = 'admin'` → 管理员
> - `role = 'user'` 时：
>   - 学号格式（纯数字 8-12 位 或 X 开头）→ 学生
>   - 工号格式（T 开头 或 数字 2-6 位）→ 教师

#### 图书表 (books)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| title | varchar | 书名 |
| author | varchar | 作者 |
| publisher | varchar | 出版社 |
| isbn | varchar | ISBN |
| description | text | 简介 |
| cover_url | varchar | 封面图片 URL |
| category | varchar | 图书分类 |
| total_copies | int | 总复本数（默认 1） |
| available_copies | int | 可借复本数（默认 1） |
| created_at | datetime | 添加时间 |

> **说明**：图书状态通过 `available_copies` 动态计算：
> - `available_copies > 0`：可借
> - `available_copies = 0`：已全部借出

#### 借阅记录表 (borrow_records)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| user_id | int | 用户 ID |
| book_id | int | 图书 ID |
| borrow_date | datetime | 借阅日期 |
| due_date | datetime | 应还日期 |
| return_date | datetime | 实际归还日期 |
| status | varchar | 状态（borrowing/returned） |

## 8. 边界条件与业务规则

### 8.1 借阅规则
- 每人最多同时借阅 5 本书（借阅中状态）
- 借阅期限 30 天
- **只有当图书 `available_copies > 0` 时才能借阅**
- 借阅时：`available_copies--`
- 归还时：`available_copies++`

### 8.2 超期处理
- 超期判断：当前日期 > due_date 且 status='borrowing' 即为超期
- 超期后仍可归还，但会产生超期记录
- 有超期未还图书记录（status='borrowing' 且超期）的用户，不能借阅新书
- 超期图书归还后，借阅限制解除
- 实现方式：在用户借阅时实时计算检查，无需定时任务

### 8.3 删除限制
- 有借阅在身的用户不能删除（存在 status='borrowing' 的借阅记录）
- **有借阅记录的图书不能删除**（`available_copies < total_copies`）
- 管理员账号不能删除（操作列显示"-"）

### 8.4 权限控制
- 未登录用户只能访问登录/注册页面
- 普通用户访问管理员页面应被拒绝
- 所有写操作（借阅、归还、增删改）需要验证权限

## 附录 A：API 接口参考

### A.1 Servlet 模块划分

本系统采用**模块化 Servlet 设计**，按功能模块划分，每个 Servlet 内部通过 `action` 参数或 HTTP 方法区分具体操作。

| Servlet 类名 | 映射路径 | 负责功能 |
|-------------|----------|----------|
| `AuthServlet` | `/api/auth` | 认证相关（登录/注册/登出） |
| `UserServlet` | `/api/user` | 用户个人中心（信息/密码/邮箱） |
| `BookServlet` | `/api/books` | 图书浏览与查询 |
| `BorrowServlet` | `/api/borrow` | 借阅相关（借阅/归还/记录） |
| `AdminBookServlet` | `/api/admin/books` | 管理员 - 图书管理 |
| `AdminUserServlet` | `/api/admin/users` | 管理员 - 用户管理 |
| `AdminBorrowServlet` | `/api/admin/borrows` | 管理员 - 借阅管理 |

---

### A.2 认证模块（AuthServlet）

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| POST | `/api/auth?action=register` | `username, name, password, email` | 用户注册 |
| POST | `/api/auth?action=login` | `username, password` | 用户登录 |
| POST | `/api/auth?action=logout` | - | 退出登录 |

**请求示例 - 登录**：
```json
POST /api/auth?action=login
Content-Type: application/json
```

## 附录 B：数据库设计

### B.1 数据库说明
- 数据库名称：`library_db`
- 字符集：`utf8mb4`
- 排序规则：`utf8mb4_general_ci`

### B.2 表结构

#### 用户表 (users)
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键 ID',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名（学号/工号）',
    name VARCHAR(50) NOT NULL COMMENT '姓名',
    password VARCHAR(255) NOT NULL COMMENT '密码（加密存储）',
    email VARCHAR(100) NOT NULL COMMENT '电子邮箱',
    role VARCHAR(20) NOT NULL DEFAULT 'user' COMMENT '角色：user/admin',
    user_type VARCHAR(20) NOT NULL COMMENT '用户类型：student/teacher/admin',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    INDEX idx_username (username),
    INDEX idx_role (role)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

#### 图书表 (books)
```sql
CREATE TABLE books (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键 ID',
    title VARCHAR(200) NOT NULL COMMENT '书名',
    author VARCHAR(100) NOT NULL COMMENT '作者',
    publisher VARCHAR(100) NOT NULL COMMENT '出版社',
    isbn VARCHAR(20) NOT NULL COMMENT 'ISBN 编号',
    description TEXT COMMENT '简介',
    cover_url VARCHAR(500) COMMENT '封面图片 URL',
    category VARCHAR(50) COMMENT '图书分类',
    total_copies INT NOT NULL DEFAULT 1 COMMENT '总复本数',
    available_copies INT NOT NULL DEFAULT 1 COMMENT '可借复本数',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '添加时间',
    INDEX idx_title (title),
    INDEX idx_author (author),
    INDEX idx_isbn (isbn),
    INDEX idx_category (category)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='图书表';
```

#### 借阅记录表 (borrow_records)
```sql
CREATE TABLE borrow_records (
    id INT PRIMARY KEY AUTO_INCREMENT COMMENT '主键 ID',
    user_id INT NOT NULL COMMENT '用户 ID',
    book_id INT NOT NULL COMMENT '图书 ID',
    borrow_date DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '借阅日期',
    due_date DATETIME NOT NULL COMMENT '应还日期',
    return_date DATETIME DEFAULT NULL COMMENT '实际归还日期',
    status VARCHAR(20) NOT NULL DEFAULT 'borrowing' COMMENT '状态：borrowing/returned',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id),
    INDEX idx_book_id (book_id),
    INDEX idx_status (status),
    INDEX idx_due_date (due_date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='借阅记录表';
```

### B.3 数据字典

#### users.user_type

| admin | 管理员 | role = 'admin' |

#### users.role
| 值 | 说明 |
|------|------|

#### borrow_records.status

## 附录 F：常见问题

### F.1 开发相关问题

**Q: 密码如何加密存储？**
A: 使用 MD5 或 BCrypt 对用户密码进行加密后存储，不要明文保存密码。

**Q: 如何防止 SQL 注入？**
A: 使用 PreparedStatement 代替 Statement，参数化查询。

**Q: Session 如何配置超时时间？**
A: 在 web.xml 中配置：
```xml
<session-config>
    <session-timeout>30</session-timeout>
</session-config>
```

### F.2 部署相关问题

**Q: 中文乱码如何解决？**
A: 
1. 确保数据库字符集为 utf8mb4
2. JDBC 连接添加 `characterEncoding=utf8`
3. 设置请求/响应编码：
```java
request.setCharacterEncoding("UTF-8");
response.setContentType("text/html;charset=UTF-8");
```

**Q: Tomcat 启动报错怎么办？**
A: 
1. 检查 JDK 版本是否匹配
2. 检查端口是否被占用
3. 查看 catalina.out 日志文件
